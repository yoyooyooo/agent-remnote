按这版技能，先走 help-first。当前这台机器上的 `agent-remnote 1.3.3` 没有 `scenario` 子命令，所以先不要猜 builtin id。`scenario builtin list`、`scenario schema explain`、`scenario run --dry-run` 在当前构建上都会报 `Invalid subcommand`。

今天是 `2026-03-23`。你说“过去 2 天”时，按技能里的时间字面量应理解为 `2026-03-21` 和 `2026-03-22`，对应 `daily:past-2d`。如果你想把今天也算进去，用 `daily:last-2d`。不要用 `daily:previous-2d`。

最稳的检查顺序是这 4 步。

1. 先确认 `scenario` 面是否真的存在，避免脑补 builtin id。
```bash
agent-remnote scenario run --help
agent-remnote scenario schema explain --help
agent-remnote --json scenario builtin list
```

2. 当前构建没有 `scenario`，改用只读手工预演。先拿今天和过去两天的 DN 条目 Rem ID。这里拿到的是当天条目 Rem，和 Daily Document 容器是两层对象。
```bash
today_id=$(agent-remnote --ids daily rem-id --date 2026-03-23)
d1_id=$(agent-remnote --ids daily rem-id --date 2026-03-22)
d2_id=$(agent-remnote --ids daily rem-id --date 2026-03-21)
printf '%s\n' "$today_id" "$d1_id" "$d2_id"
```

3. 只读展开过去两天 DN 和今天 DN，先确认候选 todo Rem ID，再确认今天的落点 parent。技能里对结构检查的首选读命令是 `rem outline`。
```bash
agent-remnote rem outline --id "$d1_id" --depth 6 --format json
agent-remnote rem outline --id "$d2_id" --depth 6 --format json
agent-remnote rem outline --id "$today_id" --depth 4 --format json
```

4. 拿到候选 `todoRemId` 后，对每一条做 portal dry-run。若先挂到今天条目根下，用下面这条。若你要挂到今天某个 section，把 `parent:id:${today_id}` 换成那个 section 的 Rem ID。
```bash
agent-remnote --json portal create --to "id:<todoRemId>" --at "parent:id:${today_id}" --dry-run
```

如果你后面切到带 `scenario` 的构建，技能定义的标准 dry-run 顺序是下面这组。

1. 先列 builtin。
```bash
agent-remnote --json scenario builtin list
```

2. 再看候选 builtin 暴露的变量，确认有没有 `source_scope` 和 `target_ref`。
```bash
agent-remnote --json scenario schema explain --spec builtin:<builtin-id>
```

3. 只在变量名确认后再跑 dry-run。
```bash
agent-remnote --json scenario run builtin:<builtin-id> --dry-run --var source_scope=daily:past-2d --var target_ref=daily:today
```

两个时间字面量直接记这两个就够了：

`daily:past-2d`：在 `2026-03-23` 这天表示 `2026-03-21` 和 `2026-03-22`  
`daily:last-2d`：在 `2026-03-23` 这天表示 `2026-03-22` 和 `2026-03-23`
