按这版 skill，纯只读且最稳的顺序是：

1. 先看 help，确认这轮能不能走 `scenario` 面，也确认 flag 名。
```bash
agent-remnote scenario run --help
agent-remnote scenario schema explain --help
```

2. 再查 builtin，别猜 id。你这次要的是 portal 汇总，所以先从列表里找 portal 版 builtin。这个 skill 只显式举了 `dn_recent_todos_to_today_move` 这个 move 版示例，不能直接拿来顶替 portal 版。
```bash
agent-remnote --json scenario builtin list
```

3. 时间范围先定成这两个字面量：
- `source_scope=daily:past-2d`
- `target_ref=daily:today`

补充判断：
- `daily:past-2d` 不含今天，符合“过去 2 天汇总到今天”
- `daily:last-2d` 含今天
- `daily:previous-2d` 已退役，别用
- 除了 `source_scope` 和 `target_ref`，先不要脑补别的 `--var` key

4. 拿到 portal 版 builtin id 后，只跑 dry-run。
```bash
agent-remnote --json scenario run builtin:<portal-builtin-id> \
  --dry-run \
  --var source_scope=daily:past-2d \
  --var target_ref=daily:today
```

5. 只看 `plan.compiled_execution`。
你要确认三件事：
- source 排除了今天
- target 是 `daily:today`
- lowering 出来的是 portal 汇总，不是 move

如果你坚持纯只读，`scenario builtin install ...` 先不要跑，它会写用户场景目录。

补充一个评测结论：当前本机实测 `agent-remnote 1.3.3` 没有 `scenario` 子命令，所以这套命令属于这版 skill 里的 planned / experimental surface。按这个顺序检查时，第 1 步就能暴露出 skill 和当前运行时的版本错配。
