按这个技能的路由，最短路径先看 `explain` 的 help，再直接对文件做 `schema explain`。`apiBaseUrl` 模式不影响这一步，`scenario schema *` 仍在本地执行，不会转发到 Host API。

```bash
agent-remnote scenario schema explain --help
agent-remnote --json scenario schema explain --spec "@$HOME/.agent-remnote/scenarios/recent-review.json"
```

如果你还想先做一层 authoring 校验，再补这一条：

```bash
agent-remnote --json scenario schema validate --spec "@$HOME/.agent-remnote/scenarios/recent-review.json"
```

先别直接进 `scenario run`。等 `explain` 返回了 var 名、类型、默认值，再决定是否组 `scenario run <spec> --var key=value`；真要试跑，先加 `--dry-run`。
