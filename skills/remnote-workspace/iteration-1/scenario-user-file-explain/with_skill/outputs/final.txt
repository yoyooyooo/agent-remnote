按这个技能的路由，最短路径先看 `explain`，先别直接进 `scenario run`。

```bash
agent-remnote scenario schema explain --help
agent-remnote --json scenario schema explain --spec "@$HOME/.agent-remnote/scenarios/recent-review.json"
```

只想先看它支持哪些 vars，这第二条就够了。你现在是 `apiBaseUrl` 模式，这一步也仍然本地执行，不会转发到 Host API。`explain --spec` 会列出这个 user package 暴露的 vars、类型、默认值，以及后续该传哪些 `--var key=value`。

如果你还想补一层 authoring 校验，再加：

```bash
agent-remnote --json scenario schema validate --spec "@$HOME/.agent-remnote/scenarios/recent-review.json"
```

等 `explain` 确认完参数，再决定是否跑 `scenario run`。真要试跑写入型场景，先加 `--dry-run`。
